Simplistic Interaction

There are two different designs:

  1. AR space

  2. Onscreen space

These two designs can be used interchangeably depending on the user's preference. To make the interaction with the framework even more straightforward and user-friendly, programs can be developed by using a method called Programming by Demonstration (PbD). This method simplifies the traditional robot programming by codes which makes it easier for a complete beginner or a user that has little programming skills to develop programs for the robot.

Programming by Demonstration

Robot programming by demonstration is a technique for teaching a robot how to perform tasks instead of programming it through codes. This is intended to help users that do not have the programming skills (or the time) to program the robot. Demonstrating how to accomplish the task with examples thus enables one to learn skills without specifically programming each detail. An AR environment for PbD was used in our project. Here, path following operations were taught by kinesthetics teaching. Thus, a tracker labelled sphere (target sphere) was used as a demonstration tool.

Steps involved:

  1. Move the target sphere to a desired position.
  2. Click the ‘Move Robot To’ button.
  3. Click on the ‘Make’ button.
  4. Code is compiled and waypoints are created.
  5. Repeat step 1- 4 to create more waypoints.

The coordinates of the waypoints are displayed on the AR DoBot code panel.
